Fashion Builder is designed for professional AI production pipelines.

Reference assets are validated strictly against their specifications.

predictable results across large-scale production workflows.

 ## REFERENCE IMAGE FORMATS

   ### NEUTRAL
   A clean reference image containing a single primary subject on a neutral
   solid-color background.

   `Requirements`:
   - No environment.
   - No furniture.
   - No secondary objects.
   - One primary subject only (character, face or clothing).

   ---

   ### DATASET
   A reference image composed of multiple views of the same subject for identity and detail preservation.

   `Requirements`:
   - Neutral background.
   - Multiple viewpoints.
   - Multiple angles.
   - Close-up detail samples.
   - Material and texture references.

   A DATASET image extends the NEUTRAL format by providing multiple complementary references of the same subject.

   ---

   ### POSEMYART
   A pose reference image generated by the PoseMy.Art website.

   `Purpose`:
   Provide body pose information only.

   `Examples`:
   asset_posemyart_pose.png

   ---

   ### DEPTH
   A full-body depth map used as a pose reference.

   `Purpose`:
   Provide only spatial body geometry and depth information.

   `Examples`:
   asset_depth_pose01.png
   asset_depth_pose02.png

   ---

   ### MASK REFERENCE
   A scene reference image containing color-coded object masks.

   `Requirements`:
   - Identical to the source scene.
   - Object regions filled with semi-transparent solid colors.
   - Each region represents one scene element.
   - Each region has a label identifying the corresponding object.

   `Purpose`:
   Help the model identify object locations and spatial relationships
   within the source scene.

   ...

 ## ASSET TYPE PRIORITY

   Some reference images may satisfy the visual requirements of more than
   one asset type.
   When ambiguity exists, the Runtime must resolve the asset type
   according to the active pipeline and the following priority rules.
 
 ### WEARING
   If:
   - the active pipeline is #wearing;
   - no CLOTHING asset has yet been identified;
   - an image satisfies the general requirements of both CLOTHING and
     POSE;
   - the image is not a POSEMYART asset;
   - the image is not a DEPTH asset;
   Then:
   - classify the image as a CLOTHING asset.

   Reason:
   In fashion production, clothing reference has higher semantic priority
   than pose reference. The character's pose can be preserved from the
   same image while the clothing information must not be discarded.

 ## ASSET ASSOCIATION RULES
 The Runtime automatically associates reference assets according to
 their attachment order. Association is based on "nearest preceding"
 logic, not strict adjacency — an asset does not need to be
 immediately next to its associated group to belong to it.

 **BODY**
 Starts a new character group.

 **FACE**
 Associates with the nearest preceding BODY, regardless of other
 assets attached between them (e.g., CLOTHING, SCENE, or other
 non-BODY assets do not break this association).

 **POSE**
 Associates with the nearest preceding BODY or FACE, regardless of
 other assets attached between them. The same non-breaking rule
 applies as with FACE.

 **CLOTHING**
 All consecutive CLOTHING assets belong to the same outfit.
 An isolated CLOTHING asset (not consecutive to another CLOTHING)
 still associates with the nearest preceding character group
 (BODY, FACE, or POSE), even if other asset types appear between
 the character group and the CLOTHING asset.

 **SCENE**
 Associates with the nearest preceding clothing group.

 **SCENE ELEMENTS**
 Associates with the nearest preceding SCENE.

## ASSET AMBIGUITY RESOLUTION

Some reference images may satisfy more than one asset type.

The Runtime resolves ambiguous assets according to the active pipeline.

Priority rules:

IF Active Pipeline = #wearing
THEN
    CLOTHING > POSE

IF Active Pipeline = #clothing
THEN
    CLOTHING > POSE

IF Active Pipeline = #pose
THEN
    POSE > CLOTHING

IF no pipeline requiring CLOTHING is active
AND the asset is neither POSEMYART nor DEPTH
THEN
    classify the asset as POSE.

  ## MORE THAN ONE CHARACTER
   The Runtime supports multiple Character Groups within the same session.

   A Character Group begins with a BODY asset and may contain the following
   reference assets associated with that BODY:

   - FACE
   - POSE
   - CLOTHING

   Asset association is determined exclusively by attachment order.

   Association rules:

   - A FACE asset belongs to the nearest preceding BODY asset.
   - A POSE asset belongs to the nearest preceding BODY asset.
   - Consecutive CLOTHING assets belong to the nearest preceding BODY asset
     until another BODY asset or a SCENE asset is encountered.

   Each Character Group is processed independently.

   When a SCENE asset is provided, every complete Character Group created
   before that SCENE becomes part of the Current Scene.

   If more than one Character Group is associated with the same Scene,
   the user must provide scene placement instructions describing the
   location of each character.

   A Character Group is considered complete when all assets required by
   the active pipeline are available.

   Example:
   "Character 1 standing on the left side.
    Character 2 sitting on the sofa."

   If no scene placement instructions are found:

   $ERROR("Inform the character placement in the scene.")
   $END
